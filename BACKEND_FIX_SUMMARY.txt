│  Symptoms:                                                                   │
│    • "cannot find symbol" errors for getId(), getName(), getEmail()          │
│    • "cannot find symbol" errors for getPassword(), getDuration()            │
│    • "cannot find symbol" errors for getType(), getWorkoutDate()             │
│    • "cannot find symbol" errors for getSteps(), getDistance()               │
│    • "cannot find symbol" errors for getCaloriesBurned(), getNotes()         │
│    • Errors in JwtTokenProvider, AuthService, WorkoutService                 │
│    • Errors in ActivityService, GoalService                                  │
│                                                                              │
│  Root Cause:                                                                 │
│    @Data annotation doesn't always generate accessible getters/setters       │
│    in complex Spring Boot + JPA + Security contexts                          │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│                         SOLUTION APPLIED                                     │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  Replaced @Data with explicit Lombok annotations:                           │
│                                                                              │
│    @Getter              → Generates all getter methods                       │
│    @Setter              → Generates all setter methods                       │
│    @NoArgsConstructor   → Generates no-args constructor                      │
│    @AllArgsConstructor  → Generates all-args constructor                     │
│                                                                              │
│  This ensures all methods are properly generated and accessible.             │
│                                                                              │
